# Offline i18n shaping/subsetting tooling for the LVGL screens.
#
# Pinned for reproducibility: the offline HarfBuzz pre-shaping step bakes glyph
# IDs and GPOS offsets into signed packs, so a shaper bump is a *reviewed
# re-shaping event* (see docs/complex-script-shaping-spike-plan.md). The
# uharfbuzz wheel bundles its own libharfbuzz, which pins the shaping engine
# independent of any system HarfBuzz.
#
#   uharfbuzz 0.55.0  ->  bundled libharfbuzz 14.2.1
#
# fontTools provides pyftsubset (subset-then-shape) and TTFont introspection.
# Pillow (with the libraqm layout engine) renders the reference oracle PNGs the
# glyph-run render is diffed against; NumPy backs the ink-IoU comparator in
# validate_runs.py. Both are typically already present from the screenshot
# tooling, so they are listed but version-loose.
#
# PyICU exposes ICU's dictionary-aware line BreakIterator, which finds real WORD
# boundaries in the no-space scripts (Thai/Lao/Khmer/Burmese) for on-device line
# wrapping. The word dictionary lives in the SYSTEM libicu (not the wheel), so the
# boundaries depend on the system ICU version — recorded as `icu_version` in each
# pack manifest so an ICU bump is a reviewed re-segmentation event, like a
# HarfBuzz bump. Build needs libicu-dev present.

uharfbuzz==0.55.0
fonttools==4.57.0
PyICU==2.16.2
Pillow>=10
numpy>=1.24
